# TirKolija — інструкція користувача

Управлінський облік транспортного бізнесу: рейси, гроші, пальне, водії, документи, Telegram-бот. Застосунок працює в браузері (телефон чи комп'ютер), дані — у Supabase, водії працюють через бот @TirKolia_bot.

---

## 1. Перед запуском: контрольний список

Один раз, у такому порядку:

1. **SQL-міграції.** Supabase → SQL Editor → виконати по черзі вміст файлів з папки `supabase` репозиторію, яких ще не виконували. Актуальний повний список: `schema.sql` (виконано), `migration_002` … `migration_007`. Кожен файл виконується один раз; повторний запуск безпечний (нічого не зіпсує, але й не потрібен).
   - migration_003 — Telegram-бот (виконано)
   - migration_004 — точки маршруту з координатами
   - migration_005 — номер зголошення RMPD/SENT
   - migration_006 — пальне (літри, норма), валюта ставки, форма оплати фрахту
   - migration_007 — реквізити компанії для рахунків/актів
2. **Бот на актуальному коді.** Якщо після останнього передеплою `tg-bot` у GitHub з'являлись зміни в `supabase/functions/tg-bot/index.ts` — передеплоїти (процедура в розділі 10).
3. **Реквізити компанії.** Застосунок → Документи → «Реквізити компанії»: назва, ЄДРПОУ/РНОКПП, адреса, IBAN, банк, підписант, телефон, режим ПДВ. Без цього не сформуються рахунки й акти.
4. **Печатка і підпис.** Там само: два PNG з прозорим фоном. Накладаються на заявки, рахунки, акти.
5. **Норми пального.** Машини → Редагувати → «Норма пального, л/100 км» для кожної машини. Без норми блок «Пальне» показує факт, але не рахує перевитрату.

---

## 2. Довідники

### Контрагенти
«+ Додати»: тип (експедиція / вантажовідправник / СТО / перевізник / інше), назва, ЄДРПОУ (потрібен для рахунків), договірний термін оплати в днях (від нього рахується дедлайн оплати рейсу і дисципліна платежів в аналітиці).

### Машини
Назва (як зручно вам, напр. «DAF 105»), номери тягача і причепа, поточний спідометр, норма пального. Спідометр далі оновлюється автоматично, коли водій фіксує кінець рейсу в боті.

### Водії
ПІБ, телефон, схема ЗП:
- **% від фрахту** — відсоток від фрахту в грн за курсом НБУ;
- **Ставка за км** — окремо Україна/закордон, множиться на км з рейсу;
- **Ставка за рейс** — фіксована сума, можна у валюті (вибір валюти поруч), конвертується за курсом НБУ на дату розвантаження;
- **% від чистого прибутку** — відсоток від (фрахт − усі витрати рейсу);
- **Змішана** — ставка за рейс (з валютою) + % від фрахту.

«Податки» — включені у ставку чи рахуються окремо (довідково для вас).

### Редагування будь-чого
На комп'ютері — кнопка «Редагувати» в рядку. На телефоні — **затиснути рядок на пів секунди**. Працює скрізь: машини, водії, контрагенти, документи, витрати.

---

## 3. Рейс

### Створення
Рейси → «+ Додати»: режим (перевізник — своя машина / експедиція — чужий перевізник, дохід = комісія), замовник, машина, водій, маршрут, фрахт і валюта, дати.

### Точки маршруту (для карти, водія, RMPD/SENT)
У картці рейсу → Редагувати: координати завантаження і вивантаження, замитнення (місце + координати), пункт пропуску (місце + координати), розмитнення (місце + координати), номер зголошення RMPD/SENT.

Координати беруться так: у Google Maps затиснути точку → внизу з'являться числа → скопіювати → вставити в поле. Формат неважливий — при збереженні все приводиться до 5 знаків після коми (вимога RMPD/SENT).

### Що дають точки
- Кнопка **«Маршрут на карті»** — Google Maps з маршрутом через усі точки по порядку.
- **«Надіслати водію (Telegram)»** — повне повідомлення: маршрут, дати, точки з координатами, номер зголошення, посилання на карту. Якщо водій підключений до бота — летить прямо в його чат; якщо ні — відкриється вікно «поділитись».
- У боті кнопка «Мій рейс» показує те саме.

### Статуси
Ведіть статус по ходу: підтверджено → заявку підписано → завантаження → замитнення → в дорозі → розмитнення → розвантаження → чекаємо ТТН → документи відправлено → очікує оплати → оплачено → закрито. Дашборд і аналітика спираються на статуси й дати.

### Дати
Біля кожної дати є ✕ для очищення (системна кнопка «Скинути» на iPhone не працює — це вада Safari, користуйтесь ✕).

## 4. Гроші рейсу

### Витрати
У картці рейсу → «Витрати рейсу»: категорія, сума, валюта, **літри** (для пального), форма оплати, дата, примітка. Валютні суми конвертуються в грн за курсом НБУ на дату витрати автоматично. Редагування — «Ред.» або затискання рядка; видалення — ✕.

### Надходження
Розділ «Надходження» в картці: часткові оплати, форма оплати, дата. Курс НБУ по фрахту береться на дату розвантаження.

### Форма оплати фрахту
У формі рейсу: безготівка / готівка / картка — ознака для вашого обліку. Податки з фрахту вносите вручну звичайною витратою (категорії податків водію в боті не показуються).

### Витрати поза рейсом
Сторінка «Гроші» → вкладка «Витрати»: витрати на машину без прив'язки до рейсу (страховка, ремонт, стоянка). Теж з літрами, якщо це пальне.

---

## 5. Пальне

Ланцюжок: норма в машині → літри у витратах (вносить водій у боті або ви) → пробіг зі спідометра (початок/кінець фіксує водій у боті).

У картці рейсу з'являється блок «Пальне»: заправлено літрів, пробіг, **факт л/100 км**, норма, і головне — **перевитрата чи економія** в літрах і гривнях (за середньою ціною заправок цього рейсу). Перевитрата підсвічується.

---

## 6. ЗП водія

У картці рейсу → «ЗП водія»: оберіть схему (підтягується схема водія), натисніть **«Розрахувати»** — база й сума заповняться за даними рейсу (валютні ставки конвертуються по НБУ). Суму можна поправити руками → «Нарахувати». Виплату фіксуєте кнопкою «Виплачено».

---

## 7. Документи

### Документи рейсу
У картці рейсу: заявки, CMR, митні документи. Заявку-PDF можна підписати печаткою прямо тут (кнопка з вибором позиції підпису). Фото від водія з бота падають сюди ж із вибраним ним типом.

### Рахунок і акт
Картка рейсу → Документи → **«Сформувати рахунок» / «Сформувати акт»**:
- реквізити ваші (з налаштувань) і замовника (з контрагентів);
- послуга з маршрутом і номером рейсу, сума фрахту, ПДВ якщо увімкнено, сума прописом українською;
- печатка й підпис накладаються автоматично, якщо завантажені;
- документ відкривається в новій вкладці (друк / збереження звідти) і одночасно зберігається в документи рейсу — звідти його завжди можна відкрити знову і передати в бухгалтерію.

Нумерація: номер рейсу (акт — з суфіксом «-А»).

### Документи компанії
Сторінка «Документи»: виписки, техпаспорти, страховки — щоб швидко переслати замовнику. Імена файлів можуть бути кирилицею — у сховище вони кладуться під технічним іменем автоматично.

## 8. Бот для водія (пам'ятка — можна переслати водію)

**Підключення:** диспетчер у застосунку (Водії → «Код підключення» → «Копіювати») надсилає водію команду виду `/start ABC123`. Водій відкриває @TirKolia_bot і надсилає її (найкраще — скопіювати й вставити, а не набирати).

**Кнопки внизу чату:**
- 🚛 **Мій рейс** — маршрут, дати, точки з координатами, номер зголошення, посилання на карту.
- 📏 **Спідометр** — «Початок» або «Кінець» → надіслати число. Кінець рейсу автоматично оновлює спідометр машини і показує пробіг.
- 💸 **Витрата** — категорія → сума (`1500` = грн, або `550 eur`, `200 pln`, `40 usd`) → для пального ще літри → примітка (або «-»). Валюта сама переведеться в грн за курсом НБУ.

**Документи:** фото CMR, чеків, митних документів (або PDF) просто надіслати в чат → бот спитає, що це (CMR/ТТН, митна декларація, T1, інвойс, акт, інше) → документ прикріпиться до активного рейсу. Підпис під фото стане назвою документа.

**«Скасувати»** або /cancel — перервати будь-який ввід.

Активний рейс = останній рейс водія не в статусі «оплачено/закрито/скасовано».

---

## 9. Дашборд, аналітика, експорт

- **Дашборд** — поточні рейси, дедлайни оплат і ТТН.
- **Аналітика** — рентабельність по машинах і рейсах, кеш-флоу по місяцях, дисципліна оплат замовників (рейтинг A/B/C).
- **Гроші → «Експорт CSV (бухгалтерія)»** — всі операції файлом для передачі бухгалтеру чи імпорту.

## 10. Обслуговування

### Оновлення застосунку
Сайт оновлюється сам через 1–2 хв після кожного пушу в GitHub. Якщо змін не видно — закрити вкладку повністю і відкрити знову (кеш).

### Оновлення бота (після змін у index.ts)
1. Supabase → Edge Functions → `tg-bot` → Settings → Delete function.
2. Deploy a new function → Via Editor → назва точно `tg-bot`.
3. GitHub → `supabase/functions/tg-bot/index.ts` → Raw / іконка копіювання → вставити в редактор.
4. **Verify JWT — вимкнути** → Deploy.
Секрети і вебхук переживають перестворення, їх чіпати не треба.

### Секрети функції (уже налаштовані)
`TELEGRAM_BOT_TOKEN` (від BotFather), `TG_WEBHOOK_SECRET` (перевірка, що пише саме Telegram), `SB_SECRET_KEY` (ключ `sb_secret_…` для доступу до бази).

### Типові проблеми
- **Бот мовчить** → Edge Functions → tg-bot → Logs; і `getWebhookInfo` у браузері (`https://api.telegram.org/bot<ТОКЕН>/getWebhookInfo`) — поле `last_error_message`.
- **«Код не знайдено»** → водій набрав код в укр розкладці або код застарів. Надіслати через «Копіювати» з застосунку.
- **«permission denied»** у боті → у SQL Editor повторити гранти: `grant all on all tables in schema public to service_role;`
- **Дата не очищається** → кнопка ✕ біля поля.
- **Кнопки бота зникли** → іконка квадратика з крапками праворуч від поля вводу в Telegram.
